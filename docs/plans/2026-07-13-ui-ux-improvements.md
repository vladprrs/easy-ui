# План: UX-улучшения пользовательского UI — плеер и редактор в приоритете (2026-07-13)

Источник: multi-agent UX-ревью от 2026-07-13 — шесть зонных ревьюеров (плеер, редактор, галерея/библиотека, CJM/visual, app-шелл/a11y, продуктовая стратегия), каждый прочитал код своей зоны и прогнал живое приложение через Playwright (137 скриншотов, ключевые состояния, эмуляция ошибок API и узких viewport), плюс критик полноты, который перепроверил спорные находки вживую. Итог: 101 находка (2 подтверждённых P0, ~30 P1), 15 пропущенных областей от критика, 11 спорных пунктов с зафиксированными решениями ниже.

## Ключевые факты ревью

- **Главный сценарий продукта — «показать прототип заказчику» — сейчас проваливается на первом же показе.** На /p/* друг под другом ~200px хрома (белый app-header + тёмный header плеера + строка device-чипов); `DeviceFrame` масштабирует только по ширине (`src/player/DeviceFrame.tsx:29`: `scale = min(1, availW/w)`), фрейм 390×844 не влезает по высоте даже в 1440×900 — CTA прототипа всегда за фолдом. Заказчик видит всю оснастку: навигацию Gallery/Library/Visual/**Debug**, сайдбар со списком всех экранов (спойлер финала флоу), переключатель устройств.
- **Флагманское демо встречает битым визуалом и противоречивым стейтом.** `prototypes/checkout.json` ссылается на несуществующий `/images/sneakers.jpg` (SPA-fallback отдаёт 200 text/html → сломанный `<img>` с alt-текстом во всех вью: плеер, CJM, лента редактора). Критик дополнительно подтвердил: `cart-total` — захардкоженный текст «Итого: 7 990 ₽ · товаров: ${/cart/count}», т.е. при deep-link/reload экран показывает «Корзина пуста» и «Итого: 7 990 ₽ · товаров: 0» одновременно, кнопка «Оформить» активна при пустой корзине.
- **Сервер сильнее UI ровно в точке ценности.** `publishPrototype`/`listPrototypeVersions`/`listPrototypeRevisions`/`restorePrototype`/`createPrototype` уже есть в `src/api/client.ts:137-157`, но ни одна страница их не вызывает: на живой базе у всех прототипов `latestVersion=null`. Публикация, версии, история, откат, создание прототипа — curl-only. Шаринг наружу требует выдать BasicAuth от всего workspace (`server/main.ts:66-78`).
- **Редактор (шаг 1) архитектурно пригоден для конструктора, но UX-дыры делают его опасным.** Лента экранов рендерит тайлы 280px шириной без капа высоты (`src/editor/EditorScreenStrip.tsx:26-36`) → на 1440×900 канвас+инспектор видны полоской ~80px (подтверждено скриншотом, критик уточнил: на 1024×700 страница скроллится, severity P1). Тихая потеря правок при SPA-навигации (beforeunload не ловит react-router), нет undo/redo, `Number('') === 0` молча коммитит 0 (`src/editor/propsForm/PropsForm.tsx:80`), события элементов (суть кликабельного прототипа) в инспекторе невидимы, восстановление после 409 — тупик «скопируй JSON вручную».
- **Reload/deep-link молча сбрасывает стейт флоу** (`src/player/navigation.tsx:48-55`: bootstrap сохраняет screenId при flowDepth 0, store пересоздаётся) — получатель ссылки на середину флоу видит противоречивый экран без объяснения. Станет массовым со share-ссылками.
- **404 версии выдаёт «Prototype not found»** (`src/player/PrototypeLoader.tsx:57-60` маппит любой 404) — устаревшая ссылка заказчика выглядит как удалённый прототип.
- **Смесь RU/EN на каждой поверхности** (все 6 зон): «Прототипы» + «Choose a flow…», «← Галерея» + Back/Restart, «API недоступен» + Retry. `document.title` всегда «easy-ui», favicon 404.
- **Что уже хорошо (не сломать):** deep-links экранов + корректная браузерная история и sessionNonce/stale-гейт; per-screen ErrorBoundary; Hotspot — настоящая `<button>` с aria; PropsForm из zod-схем; editorReducer/docMutations — чистые и иммутабельные (готовый фундамент undo и структурных операций); CJM-тайлы с живыми компонентами и stateOverrides; graceful degradation Library без Storybook; образцовая API-поверхность (OpenAPI/render-status/screenshots).
- **Пропуски ревью, влияющие на план (критик):** мобильный браузер заказчика (≤430px) — плеер непригоден (сайдбар в пол-экрана, не сворачивается); desktop-прототип в плеере — белый лист без min-height, карточка в углу пустого полотна; custom DS (yandex-pay) в редакторе/плеере не проверялись (в dev-базе нет прототипов на них); редактор не открывался на composition-фичах (repeat/$item в ElementTree/propsForm); длинные названия и 20+ экранов нигде не проверены; UI управления ассетами не существует (при этом три зоны советуют «загрузить через /api/assets»).

## Сквозные решения (фиксируются до декомпозиции)

1. **Язык продукта — русский.** Все строки UI переводятся на RU (EN остаётся для доменных имён: CJM, Storybook, API). Заводится словарь `src/app/strings.ts` (плоский constants-модуль, без i18n-фреймворка) — новые поверхности обязаны брать строки из него. Если владелец решит выходить на EN-рынок — словарь станет точкой i18n, но это вне скоупа.
2. **Единая модель навигации по прототипу.** На всех вью /p/* один и тот же хедер: хлебная крошка «Галерея / {Имя}» + сегмент-контрол «Плеер · CJM · Редактор». Убирается разнобой «← Галерея» / «← Назад к CJM» / отсутствие возврата в CJM. Глобальная навигация на /p/* сворачивается в крошку (см. решение 4).
3. **Единая политика состояния флоу.** Стейт живёт от startScreen; deep-link/reload на середину флоу показывает ненавязчивый баннер «Состояние флоу сброшено — Начать сначала?» с Restart. stateOverrides из CJM в интерактивный плеер **не** сеются (ломает семантику Back/Restart) — вместо этого CJM-тайл получает бейдж «демо-состояние». Навигация по сайдбару/стрелкам — «авторский просмотр»: replace-переход, flowDepth не растёт, кнопка Back прототипа на неё не реагирует.
4. **Один лэйаут-редизайн канваса плеера, а не три патча.** Fit-to-height, центрирование фрейма, позиция inspector-панели и сворачиваемый сайдбар меняют одну геометрию — делаются одним пакетом (W1-1/W1-2), никаких промежуточных точечных фиксов.
5. **UI закрывает разрыв с API, а не изобретает новые механизмы.** Publish/versions/revisions/create/screenshot-job — только существующие эндпоинты; новый серверный код появляется единственно для scoped share-ссылок (W3-3) и дедупликации no-op ревизий (W2-5).
6. **Превью в галерее — клиентские inert-тайлы** (механизм CJM/ленты редактора), не серверные скриншоты: ноль серверной работы, всегда актуальны по headRev. Screenshot-API остаётся для visual regression.
7. **Спорные рекомендации ревью, которые решено НЕ делать:** force-save по умолчанию при 409 (опасен без diff — идёт вместе с diff-предпросмотром в W2-4); disable кнопки «Сохранить» при !dirty (пере-сохранение полезно; вместо этого сервер дедуплицирует no-op ревизии); авто-граф рёбер CJM вместо линейной ленты (CJM — авторская линейная карта; делаем только фикс клиппинга стрелок + бейджи нелинейных переходов); скрытие маршрута /debug из прод-сборки (используется как смоук — убираем только пункт из навигации).

### W4 — Конструктор (редактор шаг 2; XL — отдельный план исполнения после ревью этого)

Архитектурная оценка ревью: editorReducer/docMutations иммутабельны и расширяемы; introspect.ts генерит формы из zod → палитра выводится из ComponentDefinition + atomic-уровней; слабое место — DOM-based hit-test (`span[data-jr-key]`, O(n) layout-reads, фрагментные rects) — для DnD нужен кэш rects и модель дроп-таргетов из дерева spec; событий в canvas-спеке нет (stripSpecEvents), но исходный spec их хранит — редактор событий не заблокирован.
