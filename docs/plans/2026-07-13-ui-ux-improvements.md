# План v2: UX-улучшения пользовательского UI — плеер и редактор в приоритете (2026-07-13)

Источник: multi-agent UX-ревью от 2026-07-13 — шесть зонных ревьюеров (плеер, редактор, галерея/библиотека, CJM/visual, app-шелл/a11y, продуктовая стратегия), каждый прочитал код своей зоны и прогнал живое приложение через Playwright (137 скриншотов, эмуляция ошибок API и узких viewport), плюс критик полноты с живой перепроверкой спорных находок. Итог: 101 находка (2 подтверждённых P0, ~30 P1), 15 пропущенных областей, 11 спорных пунктов. Раунд 1 ревью плана (Codex gpt-5.6-sol, max): 10 blocker / 20 major / 4 minor — все приняты (2 с модификацией), триаж в конце. Этот план **замещает** `docs/plans/2026-07-10-flow-state-persist-share.md` (sessionStorage-resume и state-in-query противоречат принятой политике состояния и hardened store из feedback-плана; файл помечен superseded).

## Ключевые факты ревью

- **Главный сценарий продукта — «показать прототип заказчику» — сейчас проваливается на первом же показе.** На /p/* друг под другом ~200px хрома (белый app-header + тёмный header плеера + строка device-чипов); `DeviceFrame` масштабирует только по ширине (`src/player/DeviceFrame.tsx:29`: `scale = min(1, availW/w)`), фрейм 390×844 не влезает по высоте даже в 1440×900 — CTA прототипа всегда за фолдом. Заказчик видит всю оснастку: навигацию Gallery/Library/Visual/**Debug**, сайдбар со списком всех экранов (спойлер финала флоу), переключатель устройств.
- **Флагманское демо встречает битым визуалом и противоречивым стейтом.** `prototypes/checkout.json` ссылается на несуществующий `/images/sneakers.jpg` (SPA-fallback отдаёт 200 text/html → сломанный `<img>` во всех вью). Критик подтвердил: `cart-total` — захардкоженный текст «Итого: 7 990 ₽ · товаров: ${/cart/count}» → при deep-link/reload экран показывает «Корзина пуста» и «Итого: 7 990 ₽ · товаров: 0» одновременно, «Оформить» активна при пустой корзине.
- **Сервер сильнее UI ровно в точке ценности.** `publishPrototype`/`listPrototypeVersions`/`listPrototypeRevisions`/`restorePrototype`/`createPrototype` уже есть в `src/api/client.ts:137-157`, но ни одна страница их не вызывает: на живой базе у всех прототипов `latestVersion=null`. Публикация, версии, история, откат, создание — curl-only. Шаринг наружу требует выдать BasicAuth от всего workspace (`server/main.ts`).
- **Редактор (шаг 1) архитектурно пригоден для конструктора, но UX-дыры делают его опасным.** Лента экранов без капа высоты (`src/editor/EditorScreenStrip.tsx:26-36`) → на 1440×900 канвас+инспектор видны полоской ~80px (критик: на 1024×700 скроллится, severity P1). Тихая потеря правок при SPA-навигации (beforeunload не ловит react-router), нет undo/redo, `Number('') === 0` молча коммитит 0 (`src/editor/propsForm/PropsForm.tsx:80`), события элементов в инспекторе невидимы, восстановление после 409 — тупик «скопируй JSON вручную». Ревью раунда 1 нашло латентную потерю данных: клиентский `PrototypeDraft` отбрасывает `figma`/assets, и каждое editor-save создаёт ревизию без provenance (`src/api/client.ts:59`).
- **Reload/deep-link молча сбрасывает стейт флоу** (`src/player/navigation.tsx:48-55`) — получатель ссылки на середину флоу видит противоречивый экран без объяснения. Станет массовым со share-ссылками.
- **404 версии выдаёт «Prototype not found»** — причём сервер сам не различает случаи: `server/repos/prototypes.ts:210` возвращает общий `not_found` и для прототипа, и для версии.
- **Смесь RU/EN на каждой поверхности** (все 6 зон); `document.title` всегда «easy-ui», favicon 404.
- **Что уже хорошо (не сломать):** deep-links экранов + браузерная история + sessionNonce/stale-гейт; per-screen ErrorBoundary; Hotspot — настоящая `<button>` с aria; PropsForm из zod-схем; editorReducer/docMutations — чистые и иммутабельные; CJM-тайлы с живыми компонентами; graceful degradation Library; образцовая API-поверхность (OpenAPI/render-status/screenshots).
- **Пропуски ревью, влияющие на план (критик):** мобильный браузер заказчика (≤430px) — плеер непригоден; desktop-прототип в плеере — белый лист без min-height; custom DS в редакторе/плеере не проверялись (в dev-базе нет таких прототипов); редактор не открывался на composition-фичах; длинные названия и 20+ экранов не проверены; UI управления ассетами не существует.
- **Факты из ревью плана (раунд 1):** `useBlocker` в установленном react-router требует data-router context — при `BrowserRouter` упадёт в рендере (`node_modules/react-router/.../hooks.js:1247`), приложение сидит на `BrowserRouter` (`src/app/main.tsx:9`); `server/seed.ts` применяет JSON-фикстуру ровно один раз по `seed_log` и не грузит ассеты; повторный PUT прототипа — не no-op (репин бандлов/темы/ассетов, CAS, message); share-страница тянет субресурсы (JS/CSS, DTO, бандлы, shims, ассеты), которым нельзя передать токен-заголовок через `import()`; `upsertReference` заменяет asset_id, а `visual_runs` не хранит baseline-asset → «снять baseline» перепишет доказательства старых прогонов; `/api/assets` поддерживает только POST; каталог `src/catalog/definitions` не существует — Image приходит из `@json-render/shadcn`; canvas-превью редактора/CJM до сих пор зовут legacy `stripSpecEvents`/`splitCanvasSpec` вместо RuntimeTree-операций feedback-плана.

## Сквозные решения (фиксируются до декомпозиции)

1. **Язык продукта — русский, с явной границей.** Локализуется product chrome; authored-контент прототипов и custom-бандлы не трогаются. Словарь `src/app/strings.ts` — константы **и функции** для динамических строк (интерполяция, склонения); серверные ошибки показываются через `formatApiError(code, details)` (маппинг кодов на RU-строки), не сырым message. Allowlist доменных терминов: CJM, Storybook, API, Basic Auth, ревизия→rev в техконтексте. EN-рынок вне скоупа; словарь — будущая точка i18n.
2. **Единая модель навигации по прототипу — отдельный компонент.** `PrototypeChrome` (WF-4): хлебная крошка «Галерея / {Имя}» + сегмент-контрол «Плеер · CJM · Редактор» + версия/статус, один на все /p/*-вью; глобальный header на /p/* сворачивается в крошку. Презентация и share рендерятся **вне** него.
3. **Единая политика состояния флоу.** Стейт живёт от startScreen. Различаем три способа входа не «flowDepth===0», а явным `entryReason` в location state: `bootstrap` (deep-link/reload → баннер «Состояние флоу сброшено — Начать сначала?»), `browse` (сайдбар/стрелки: replace-переход `browseToScreen`, вне flowDepth, кнопка Back прототипа не реагирует), `flow` (навигация действиями прототипа). stateOverrides из CJM в интерактивный плеер не сеются — CJM-тайл получает бейдж «демо-состояние». Query string (в т.ч. ?debug=1) сохраняется всеми переходами плеера. План `2026-07-10-flow-state-persist-share.md` (sessionStorage-resume, state в share-query) — superseded.
4. **Один лэйаут-редизайн канваса плеера, а не три патча.** Fit-to-height, центрирование, позиция inspector-панели и сворачиваемый сайдбар — один пакет (W1-1) поверх общего модуля метрик устройств (WF-3).
5. **Канонический viewport прототипа неприкосновенен.** Декоративный bezel/статусбар — **вне** точного prototype viewport (иначе разойдутся скриншоты/CJM/capture). Размеры устройств — единый модуль `src/designSystems/deviceMetrics.ts` (WF-3), потребители: Player, Editor, CJM, Capture; политика desktop-высоты (auto/fixed) документируется там же.
6. **UI закрывает разрыв с API, а не изобретает новые механизмы.** Новый серверный код появляется только там, где без него нельзя: share-сессии (W3-3), миграция `visual_runs.reference_asset_id` (W5-4), коды ошибок `prototype_not_found`/`version_not_found` (W0-4). Каждый новый/изменённый endpoint регистрируется в `server/contracts.ts`, перегенерирует `server/openapi.json`, получает happy/error contract-тесты; миграции строго сериализуются (порядок: W0-4 → W3-3 → W5-4).
7. **Превью в галерее — клиентские inert-тайлы с перф-гейтом.** Lazy-рендер по IntersectionObserver, лимит параллельной загрузки, unmount вне вьюпорта, event-stripped RuntimeTree, error boundary на тайл; перф-критерий: 30+ карточек (в т.ч. custom DS) без деградации TTI галереи > 20%. Если бюджет не проходит — фиксируем откат на кешированные серверные скриншоты (решение по результату бенча в W5-3, не догма).
8. **Save = осознанный checkpoint, без серверной дедупликации.** Повторный PUT намеренно репинит бандлы/тему/ассеты и принимает message — «мусорных ревизий» как класса нет, глобальный dedupe менял бы публичный контракт (решение раунда 1). W2-5 сужен до Ctrl+S + фиксация semantics save в docs.
9. **Спорные рекомендации ревью, которые решено НЕ делать:** force-save по умолчанию при 409 (только вместе с трёхсторонним diff, W2-4); disable «Сохранить» при !dirty (см. решение 8); авто-граф рёбер CJM (CJM — авторская линейная карта; только фикс клиппинга стрелок + бейджи статически доказуемых navigate-переходов, динамические помечаются «динамический переход»); скрытие маршрута /debug из прод-сборки (смоук; убираем только пункт навигации).

## Волны исполнения

Каждая задача — свежий Codex-диспатч `--write --effort medium`; владение файлами внутри волны эксклюзивно; задачи, делящие файл, соединены стрелкой DAG (последовательно). Каждая задача **владеет и своими тест-файлами** (юниты/е2е, которые она заведомо ломает) и прогоняет целевые сьюты сама — не «чиним всё в конце волны».
